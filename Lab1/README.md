Вариант 13. 
Используемые устройства: ряд ключей 1х4, матрица светодиодов 1х1
Написать ассемблерную программу, которая будет зажигать светодиоды разным цветом в зависимости от выставленных ключей. Соответствие ключей-цветов следующее: ключ 1 – красный, ключ 2 – синий, ключ 3 – зеленый. Если выбирается комбинация ключей, то цвет комбинируется. Если нажимается ключ 4, нужно отключить светодиод и перейти на метку end. 

Код начинается с указания адреса ключей (0xF0000004) и адреса светодиода (0xF0000000). 
Пусть у нас тумблеры 0, 1, 2, 3.
Результаты, которые ожидаем получить: 0 - красный, 1 - синий, 2 - зеленый. 0+1 - фиолетовый, 0+2 - желтый(как?...), 1+2 - бирюзовый(ок, допустим). При 0+1+2 - Белый. Ну и тумблер 3 - гаси свет. 
После инициализации адресов переходим к чтению состояния ключей, здесь li загружает в регист t0 адрес  0xF0000004, затем lw читает байты из памяти по адресу, который как раз лежит в t0, сохраняет в t1.
Далее идет проверка нажат ли 3 ключ, так как я его назвал "гаси свет" (я думаю понятно и объяснять не стоит почему), следовательно он отвечает за отключение нашего светодиода.
Команда bne сравнивает реги t2 и t3 (t2 - сохраненный результат побитового умножения маски, а t3 - addi загружает 0). Если они равны ключ 3 - не нажат. Если они равный,то ключ 3 - нажат, следовательно "Гаси свет пора спать"
Затем идет нудная проверка всех наших ключей, которые отвечают за цвета. Если кратко объяснять, то алгоритм примерно такой же, что и с 4 ключем. Есть два рега и мы их сравниваем. 
На примере синенького расскажу: у нас получается, что в регистр t2 командой addi загружается маска 2, что в двоичном виде представляет собой число 0010, то есть выделяется второй бит.
Командой and выполняется побитовое умножение состояния ключей из регистра t1 на эту маску, и результат сохраняется обратно в t2. Если второй ключ был нажат, результат будет равен 2, если не нажат — 0.
Затем в регистр t4 командой addi загружается ноль для последующего сравнения. Команда bne сравнивает регистры t2 и t4. Если они не равны, то есть ключ 1 нажат, программа прыгает на метку add_blue, где к итоговому цвету добавляется синяя компонента. 
Если же они равны, значит ключ 1 не нажат, и программа с помощью команды j переходит на метку check_green, чтобы продолжить проверку третьего ключа(2).
Аналогично идет работа для зеленого и красного. Ну и в конце у нас, если при проверке находит ключ 3(тип он нажат), то у нас программа прыгает на turn_off.
Командой sw этот ноль записывается по адресу светодиода, и светодиод гаснет. После этого программа попадает на метку end, где находится команда j end, то есть безусловный переход на саму себя.
Это создаёт бесконечный цикл, внутри которого программа ничего не делает и больше не проверяет состояние ключей.
